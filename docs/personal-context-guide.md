# 个人文档与工作上下文指南

> 本文介绍 Brain OS 中 `06-PERSONAL-DOCS/` 和 `07-WORK-CONTEXT/` 这两个目录的定位、使用方法，以及它们与整体知识库架构的关系。

---

## 为什么有 06 和 07？

Brain OS 的 vault 目录有一个核心设计原则：**不同类型的信息用不同的容器管理**。

| 目录 | 定位 | AI 访问方式 |
|------|------|------------|
| `00-INBOX/` | 灵感入口、待处理 | AI 驱动清理 |
| `01-PERSONAL-OPS/` | 个人事务驾驶舱 | AI 主动维护 |
| `03-KNOWLEDGE/` | 领域知识沉淀 | AI 可自由引用 |
| `05-PROJECTS/` | 项目注册层 | AI 读取上下文 |
| **`06-PERSONAL-DOCS/`** | **私密个人文档** | **按需访问（AI 默认不读）** |
| **`07-WORK-CONTEXT/`** | **工作经验积累** | **AI 处理工作时引用** |

06 和 07 解决的是同一个问题：**上下文积累**。

> AI 每次会话都是空白启动的。它之所以能了解你，是因为有文件记录了你的经验、决策、偏好和历史。06 和 07 就是这套"上下文记忆"的存储层。

---

## 06-PERSONAL-DOCS：私密个人文档区

### 是什么

存放**只属于你自己的敏感信息**：财务、健康、人际关系、人生目标、学习记录。

### 核心特性

- **默认不进入 nightly pipeline**：AI 不会主动处理或分析这里的内容
- **默认不进 Git 仓库**：已加入 `.gitignore`，不会推送到公开仓库
- **按需访问**：只有你明确告诉 AI "看一下 06-PERSONAL-DOCS/02-HEALTH/xxx.md"，它才会访问

### 目录结构

```
06-PERSONAL-DOCS/
├── 01-FINANCE/          # 财务：收支、预算、账单
├── 02-HEALTH/           # 健康：就医、体检、运动计划
├── 03-RELATIONSHIPS/    # 关系：重要联系人备忘、人际关系图
├── 04-LIFE-GOALS/       # 人生目标：长期愿景、年度 OKR
└── 05-LEARNING/         # 学习：课程进度、书单、读书笔记摘要
```

### 与其他目录的关系

```
06-PERSONAL-DOCS/          ← 原始私密信息存放
    ↓（用户主动提取）
03-KNOWLEDGE/              ← 去敏感化后的知识沉淀
    ↓（AI 引用）
01-PERSONAL-OPS/           ← 事务驾驶舱（个人计划、待办）
```

**注意**：不要把私密信息（如具体财务数字、就医记录）放进 `03-KNOWLEDGE/`。去敏感化后的洞察和总结可以。

### 使用建议

- 文件命名：`YYYY-MM-DD-topic.md`（便于时间排序）
- 条目写法：简洁、事实性，便于未来查找
- 定期 review：每季度整理一次，把过期内容归档

## 07-WORK-CONTEXT：工作经验积累区

### 是什么

存放你在**特定工作场景中积累的上下文**：客户信息、重大决策、成功模式、失败教训、关键干系人。

这是 Brain OS 的"职业记忆"层。

### 与 03-KNOWLEDGE 的区别

| 维度 | `03-KNOWLEDGE/` | `07-WORK-CONTEXT/` |
|------|----------------|-------------------|
| 内容类型 | 通用知识、方法论、行业原理 | 你的具体经验、你的具体决策 |
| 适用范围 | 任何人都能用 | 只对你有意义 |
| 示例 | "如何做用户访谈" | "客户 A 的访谈记录：偏好快速原型，不喜欢文档" |
| AI 访问 | 自由引用 | 工作相关时引用 |

> 可以把 `03-KNOWLEDGE/` 理解为"你学到的世界规律"，`07-WORK-CONTEXT/` 理解为"你在你的世界做过什么"。

### 目录结构

```
07-WORK-CONTEXT/
├── 01-CLIENTS/             # 客户信息、关系备忘、合作历史
├── 02-DECISIONS/           # 重大决策记录（背景 + 选项 + 理由 + 结果）
├── 03-SUCCESS-PATTERNS/    # 你的成功模式（什么方法对你有效）
├── 04-FAILURE-LESSONS/     # 失败教训和反思（不掩盖，认真记）
└── 05-STAKEHOLDERS/        # 关键干系人备忘（偏好、禁忌、关系）
```

### 与 Nightly Pipeline 的关系

07-WORK-CONTEXT 不是 pipeline 自动处理的对象，而是**手动维护的上下文库**。

建议在以下时机更新：
- **重大决策后**：决策了什么、为什么、预期结果
- **项目复盘后**：什么有效、什么失败、下次怎么做
- **重要客户/干系人变化后**：偏好变了、关系升温了、有新的雷区

## 三层上下文架构总结

Brain OS 的上下文积累是三层的：

```
┌─────────────────────────────────────┐
│  03-KNOWLEDGE/                      │  通用知识层（AI 可自由引用）
│  "我知道的世界"                       │
└─────────────────────────────────────┘
┌─────────────────────────────────────┐
│  07-WORK-CONTEXT/                   │  工作上下文层（AI 处理工作时引用）
│  "我在工作中做过的事"                  │
└─────────────────────────────────────┘
┌─────────────────────────────────────┐
│  06-PERSONAL-DOCS/                  │  个人私密层（AI 默认不访问）
│  "只属于我的信息"                      │
└─────────────────────────────────────┘
```

越靠上，AI 访问越主动；越靠下，越需要你主动授权才能访问。

---

## 开始使用

1. **先在 06-PERSONAL-DOCS 放一件事**：比如一个生活目标，或一条财务备忘
2. **在 07-WORK-CONTEXT 放一个决策**：最近做的一个重要决定，记录背景和理由
3. **下次处理相关任务时**，主动告诉 AI："可以参考 07-WORK-CONTEXT/02-DECISIONS/"

慢慢积累，上下文越丰富，AI 越了解你。
