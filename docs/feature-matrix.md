# Brain OS Feature Matrix (SSOT)

> Chinese version: [docs/zh/feature-matrix.md](zh/feature-matrix.md)

This page is the single source of truth for what Brain OS can do, which repository assets implement each capability, and how an AI agent can check whether a deployment has installed it.

Use it when you want to:

- install Brain OS incrementally instead of copying the whole repo
- ask an agent to compare a local deployment with the public framework
- decide which features are missing from your setup
- understand which features are core, optional, or advanced

---

## Install Tiers

| Tier | Meaning |
|---|---|
| **Core** | Recommended for almost every Brain OS deployment |
| **Standard** | Useful once the basic vault + agent loop works |
| **Advanced** | Powerful, but enable only after one successful workflow |
| **Optional** | Domain-specific or integration-specific |
| **Maintainer** | Mostly for repo maintainers / contributors |

## Agent Audit Prompt

Use this prompt with your local AI agent when you want an installation gap analysis:

```text
Read docs/feature-matrix.md in the Obsidian Brain OS repo.
Compare every feature row against my local deployment.
For each row, report one of: installed, partially installed, missing, intentionally skipped, or blocked.
Use the "How an agent checks it" column as the verification method.
Do not install optional or advanced features without asking.
Do not claim QMD is bundled with OpenClaw; verify QMD separately.
Return a prioritized install plan with Core gaps first, then Standard, then Optional/Advanced.
```

## Maintenance Rule

When adding a new user-facing feature to Brain OS, update this file in the same PR:

1. add one row to the matrix
2. link the main assets
3. describe how to enable it
4. describe how an agent can verify it
5. update the Chinese version as well

If a feature is internal-only and not part of the public framework, do not list it here.
