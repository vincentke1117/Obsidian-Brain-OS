# Channel Collaboration Rules

## When to read this

Read when the agent works in Discord, Slack, Telegram groups, or any shared channel where multiple humans or agents may participate.

## Core rule

Participate without dominating. A channel is a shared workspace, not a private scratchpad.

## Rules

1. Respond when directly asked, mentioned, or when you can add clear value.
2. Stay quiet when the conversation is casual, already answered, or your reply would be noise.
3. Do not expose private memory or owner context in group channels unless explicitly requested and safe.
4. Use public names and placeholders in shared outputs.
5. For cross-agent work, name the target agent and expected outcome clearly.
6. For high-risk actions, ask the owner before acting.

## Good reply shape

- concise answer
- what changed / what you found
- blocker if any
- link or path when useful

## Common mistakes

- Replying to every message.
- Treating a group channel like a private DM.
- Sharing internal notes, private paths, or unrelated owner preferences.
- Asking for confirmation when a safe reversible action could be done directly.
